API·SSE·UI 상태 정합성

Event identity와 detail/collection 갱신 경계를 검토한다.

60초 요약

EventBus 숫자 ID는 서버 재시작 때 재사용되지만 브라우저 dedup Set은 유지된다. 열린 탭에서 restart가 발생하면 기존 ID 구간의 새 event가 drop될 수 있다. Team/Hook handler는 detail 또는 열린 runs만 갱신해 collection card가 stale일 수 있다.

사용자·도메인 행동

B-01/B-02

EventSource가 EventBus history/live event를 받고 숫자 ID Set으로 중복을 거른다.

B-03/B-04

Team event는 selected detail, Hook event는 toast/badge와 열린 run 목록을 갱신한다.

현재 구조

결정 질문: restart와 domain event 후 무엇이 drop/stale되는가?

EventBus A → id 1..N → Browser seen-ID Set
server restart
EventBus B → id 1 → Browser: duplicate ID로 drop
Team/Hook event → selected detail/open runs
                - - collection row remains stale

실제 event와 state 흐름을 표현한 텍스트 fallback이다.
