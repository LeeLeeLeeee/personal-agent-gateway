요약행동현재 구조선택지근거수정 요청한계

운영 정책 적용 범위

Chat, Job, Schedule, Team Run, Hook의 Intake, Emergency Stop, Health, Audit 배선을 검토한다.

60초 요약

Background executor는 하나의 lifespan에서 시작되지만 운영 정책은 도메인별 목록으로 반복된다. Hook은 Intake/Emergency Stop, readiness, mutation audit 일부에서 빠져 있다. 현재 구조를 유지하면서 Hook 로컬 배선을 보완하는 선택을 비교한다.

실제 운영에서 Hook의 필수도와 health 503 정책은 확인되지 않았다.

사용자·도메인 행동

B-01/B-02

Emergency Stop이 새 실행을 차단하고 active 작업을 중단한다. HookLoop는 외부 이벤트를 polling해 HookRunner로 전달한다.

B-03/B-04

Health가 실행 component 생존을 알리고, 중요 mutation은 actor와 correlation ID를 audit에 남긴다.

현재 구조

결정 질문: 어떤 background 실행기가 운영 정책 경계 밖에 있는가?

Execution APIs → IntakeGate → Chat / Job·Schedule / Team Run
Hook run-now ── bypass ──→ HookRunner
HookLoop ────── bypass ──→ HookRunner
EmergencyStop → Chat / Job / Team Run
HealthService → Worker / Scheduler
EmergencyStop, HealthService - - no edge - - HookRunner

Mermaid를 실행하지 않는 환경을 위한 텍스트 fallback이다. 점선 의미는 분석 문서의 확인된 bypass 또는 누락 edge와 같다.

선택을 위한 비교

F-01 Hook 실행 중단

OptionBenefitCostRiskApplicable when
O-01/A 유지변경 없음예외 문서화Stop 후 실행Hook이 kill switch 밖인 경우
O-01/B 로컬 배선기존 Gate 재사용상태·테스트 추가재처리 정책 오류현재 단일 프로세스
O-01/C registry향후 누락 방지재설계과도한 추상화executor가 계속 증가

Recommendation: O-01/B. 세 번째 신규 executor가 추가되거나 누락이 재발하면 O-01/C를 재검토한다.

F-02 readiness

OptionBenefitCostRiskApplicable when
O-02/A 유지health 계약 불변Hook 관측 별도 필요죽은 task 미탐지Hook 비필수
O-02/B Hook lifecycle 추가현재 component 직접 관측상태·테스트 추가provider 장애와 task 사망 혼동현재 lifespan 구조
O-02/C health registryexecutor 확장 일관성등록 규약 재설계현재 규모에 과함component가 계속 증가

Recommendation: O-02/B. Hook이 선택 기능이고 503이 재시작을 유발하면 degraded detail로만 노출한다.

F-03 audit

OptionBenefitCostRiskApplicable when
O-03/A 실행 API만 유지audit volume 최소예외 정책 문서화설정 변경 추적 공백실행만 중요 event
O-03/B 위험 기반 확대중요 Hook 변경 추적endpoint별 명시 코드새 mutation 누락현재 명시적 router 구조
O-03/C 모든 mutation 자동 audit포괄성before/after·redaction 설계저가치·민감 데이터 증가전 mutation 추적 의무

Recommendation: O-03/B. 모든 변경의 법적 추적 의무가 확인되면 명시적 event 목록을 먼저 정의한다.
